Backtest / A/A Simulation

Repeated A/A splits on historical data → empirical FPR. If FPR drifts away from the nominal α, the test breaks its assumptions (heavy tails, clustering, dependencies).

Configuration

Switchback Design

When per-user splits aren't possible (marketplaces, matching, pricing) — alternate A/B by time blocks. Account for carryover and autocorrelation.

Plan Blocks

Good switchback rules: blocks ≥ system feedback time (car-driver match rate, price recomputation lag); discard the first N minutes of each block (carryover); for analysis use cluster-robust SE or block-bootstrap, otherwise SE is underestimated and FPR explodes.

Variance Reduction: CUPED / CUPAC

CUPED subtracts the correlation between Y and a pre-experiment covariate X: Y' = Y − θ·(X − E[X]), where θ = Cov(Y,X)/Var(X). CUPAC is the same idea, but X is an ML model prediction.

CUPED Adjustment

Pro tips: compute θ on pooled data (A+B) to avoid bias. Effectiveness ≈ 1−ρ²(Y,X). For ρ=0.5 variance drops by 25% → same power with 25% fewer users. CUPAC: train a gradient boosting model on historical Y without treatment and use the predictions as X.

SRM — Sample Ratio Mismatch

χ² test that the actual split matches the expected one. If p-value drops below ~0.001, your experiment is broken (data loss, bot filtering, bucketing bug). Stop the test.

Check Split

Threshold: industry standard is p < 0.0005 or p < 0.001. Not 0.05, because with hundreds of experiments false alarms would burn out your team. Check SRM every day, not just at the end.

Sequential Testing

SPRT and mSPRT let you "peek" at the experiment without inflating FPR — required for early stopping without a peeking penalty.

SPRT (Wald) — Proportion

Stopping rules: accept H₁ if LLR ≥ log((1−β)/α); accept H₀ if LLR ≤ log(β/(1−α)); otherwise — continue. Expected sample size is typically ≈ 50% of fixed-horizon.

Ratio Metrics (Delta Method)

For metrics of the form R = ΣX / ΣY (revenue per user, CTR per session), the mean of user-level ratio is biased and SE is wrong. Use the delta method.

Inputs

Delta method: Var(N/D) ≈ (1/D̄²)·Var(N) + (N̄²/D̄⁴)·Var(D) − 2(N̄/D̄³)·Cov(N,D). For clustered data (sessions within users) you need cluster-level variance.
